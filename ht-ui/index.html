# 组件规范

1、所有组件使用 SFC形式编写

2、SFC 中的标签顺序推荐使用顺序为 script、template、style，这样更符合先声明，后使用的逻辑

3、组件起名规范：起名尽量使用完整单词而不是缩写，父组件紧密耦合的子组件，命名要以父组件名为前缀、使用连字符连接单词来

# 属性相关

1、defineProps、defineEmits 要用类型声明的方式，而非运行时声明；

2、props 声明时使用驼峰，但在模板中使用时使用 连字符

3、一定要为 v-for 设置 key，且尽量不要用 index；

4、v-if 和 v-for 禁止同时出现在一个组件上；

5、推荐使用指令缩写，即:、@、#，除了 v-bind

# 其他

1、 一个SFC文件尽量不要超出300行,当超出300行时，应该考虑封装部分内容组件

2、 scoped 中尽量避免使用元素选择器

3、函数名统一用动词开头、参数控制在 3 个（含）以内

# 包管理规则

内网开发，组件不允许发到外网上去，可以使用现成的包
